## Aplicando alterações

## 1. Controle de Versão (Pages)
* **Duplique, não substitua:** Antes de alterar, duplique a página no Figma (botão direito na Page > *Duplicate*).
* **Nomenclatura:** Identifique as páginas claramente como `Web - V1`, `Web - V2`, etc. Isso cria um "de-para" que permite comparar a evolução e resgatar ideias descartadas.
* **Limitação Starter:** Em contas gratuitas (limite de 3 páginas), organize-se colocando estudos em uma página, interfaces em outra e o entregável/handoff na última.

## 2. Componentes e Bibliotecas
* **Cuidado ao deletar:** Não apague componentes "pai" usados na V1 se houver chance de voltarem na V3. 
* **Limpeza Final:** Só faça uma "limpa" nos arquivos inúteis após a aprovação final (ex: V3 aprovada), para garantir que o desenvolvedor não se perca com versões obsoletas.

## 3. Comentários como Documentação
* **Pós-Pesquisa:** Após analisar os vídeos/testes, use a ferramenta de **Comentários (tecla C)** diretamente no Figma para marcar os pontos exatos de melhoria.
* **Foco no Teste:** Não comente durante a pesquisa ao vivo; foque na reação do usuário. Deixe para pontuar no layout após a análise fria dos dados.

## 4. A "Linha do Tempo" do Projeto
* **Storytelling:** Manter o histórico (V1 + Pesquisa > V2 + Refinamento > V3) cria uma linha do tempo. 
* **Valor Profissional:** Recrutadores e clientes valorizam ver o *porquê* das mudanças. Isso prova que o design foi guiado por dados e testes, não apenas por estética.
