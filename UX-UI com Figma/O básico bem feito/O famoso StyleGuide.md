## O famoso StyleGuide

## 1. O que é o Style Guide
* **Documentação Técnica:** Registro visual do que foi criado (grids, cores, fontes) para facilitar o trabalho do desenvolvedor (*handoff*).
* **Organização:** Centraliza os elementos em um frame separado para não "sujar" o arquivo de design principal.

## 2. Estrutura do Documento
* **Grids:** Registro das malhas **Desktop (1440px)** e **Mobile (360px)**.
* **Cores:** Amostras organizadas com **Auto Layout** (Roxos, Amarelos e Pretos).
* **Assets:** Agrupamento de componentes, imagens de produtos e botões em um só lugar.

## 3. Entrega (Handoff)
* **Documento Vivo:** Deve ser atualizado até o fim do protótipo.
* **Nomenclatura:** Essencial para que a equipe técnica identifique cada estilo rapidamente.
