## Hora de testar a navegação

## 1. Configurações de Fundo (Background)
* **Dica Visual:** Ajuste a cor de fundo do protótipo (no painel *Prototype*) para uma cor próxima à do seu projeto. Isso evita bordas brancas ou contrastes agressivos que distraem o usuário.
* **Arestas:** Manter o fundo personalizado evita que monitores *ultrawide* ou de resoluções diferentes mostrem cortes secos nas laterais.

## 2. Atalhos de Escala (Tecla Z)
* **Ajuste de Tela:** Use a tecla **"Z"** no modo de visualização para alternar entre os modos de exibição.
* **Fit to Screen:** O atalho ajusta o projeto automaticamente ao tamanho da janela do navegador, corrigindo problemas de projetos desenhados em resoluções maiores ou menores que o monitor atual (ex: desenhado em 1440px e visualizado em 1280px).

## 3. Uso de Devices (Mockups)
* **Desktop:** O instrutor não recomenda o uso de *devices* (como o frame de Macbook) para desktop, pois se a resolução do desenho for diferente da do mockup do Figma, a interface pode ficar desalinhada ou com barras pretas.
* **Mobile:** O uso de *devices* (iPhone, Android) é útil apenas quando não é possível testar diretamente no aparelho físico, ajudando a dar contexto ao cliente.
* **Custom Size:** É possível configurar uma resolução personalizada para o frame do protótipo caso os presets não atendam.

## 4. Boas Práticas
* Sempre teste a escala (**Z**) antes de uma apresentação para garantir que nada foi cortado.
* Desenhe em resoluções padrão (como **1440px** ou **1920px**) para facilitar o ajuste automático do motor do Figma.
